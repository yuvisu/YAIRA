# Stage 12 — Writing

**AI role**: Co-author
**Status**: not-started

## Purpose

Produce a paper draft (or thesis chapter) by stitching together earlier stage docs. AI assembles first cuts; the human edits paragraph by paragraph and owns the final manuscript.

## Inputs

- Stage 01 (question, hypotheses) → Introduction.
- Stage 02 (lit review) → Related Work.
- Stage 03 (study design) → Methods §1 (study design).
- Stage 06 (preprocessing) → Methods §2 (data).
- Stage 08 (methods) → Methods §3 (proposed approach).
- Stage 09 (settings) → Experiments §1 (setup).
- Stage 07 + Stage 10 (baselines + results) → Experiments §2 (results).
- Stage 11 (discussion) → Discussion + Conclusion.

## Activities

1. Decide target venue and length budget. Record below.
2. Set up `artifacts/paper/` with the template (LaTeX class file from venue, or Markdown + pandoc).
3. AI drafts each section by stitching from the relevant stage docs (table below). Each section starts as a "fast first pass"; do not block on polish.
4. Human edits paragraph by paragraph; AI offers tightening, citation suggestions, and consistency checks on demand.
5. Build the bibliography from `refs/` (BibTeX).
6. Render figures and tables into the paper from `artifacts/`.
7. Pre-submission checklist (below).

## Artifact(s)

- `artifacts/paper/main.tex` (or `main.md`).
- `artifacts/paper/refs.bib`.
- Compiled PDF in `artifacts/paper/build/`.

## Pre-submission checklist

- [ ] All sections drafted and human-edited.
- [ ] Every figure/table is generated by code in this repo (no hand-edited PNGs).
- [ ] Every numerical claim in the text is traceable to `artifacts/tables/`.
- [ ] BibTeX entries match the survey table in `refs/survey.csv`.
- [ ] All abbreviations defined on first use.
- [ ] Reproducibility statement included (env, seeds, data access).
- [ ] Ethics / IRB statement included if applicable.
- [ ] Author list, affiliations, acknowledgements finalized.
- [ ] Anonymization done for double-blind venues.
- [ ] Co-authors have signed off.
- [ ] Final PDF compiles without warnings.

## Exit checklist

- [ ] `artifacts/paper/build/main.pdf` exists and matches the venue's format.
- [ ] Pre-submission checklist above is complete.
- [ ] Submission record (date, system, ID) noted in `capture.md`.
